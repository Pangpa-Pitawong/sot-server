Chapter One

Game Overview

ภาพรวมเกม
⚔️
Core Concept
เกมกระดานดิจิทัล Turn-Based ที่ผสาน กลยุทธ์ + ไพ่ + การซ่อนตัวตน ในโลกแฟนตาซีสไตล์ Stickman-Ink Anime ผู้เล่น 3–6 คนจะต่อสู้แย่งชิงบัลลังก์โดยไม่รู้ว่าใครคือ "คนทรยศ"
🎴
Unique Selling Points
  • ระบบ Hidden Role + Betrayal ที่พลิกเกมได้ตลอด
  • Art Style Stickman Ink Animation ที่ไม่มีใครทำ
  • Card + Dice + Board ใน engine เดียว
  • Evolution Class System 18 คลาส
  • เล่นได้ทั้ง Local และ Online 6 คน
🏆
Genre Inspiration
Root (Board Game) Coup Slay the Spire Among Us Fire Emblem Legends of Kingdom Rush
🎨 Art Direction: ใช้สไตล์ Stickman Ink-Sketch Animation — ตัวละครเป็น Stick Figure วาดด้วยลายเส้นหมึกดำ มีรายละเอียดเพิ่มด้วยสีน้ำ (watercolor wash) ดังเห็นจากภาพ Concept Art ที่ผู้พัฒนาอัปโหลด — สไตล์นี้ตัดกับตลาดเกมปัจจุบันและใช้ต้นทุน Production ต่ำกว่า Full 3D อย่างมาก

📈 Progression System

1

EXP Accumulation

ได้ EXP จากการโจมตี, ทำ Quest, ใช้ไพ่พิเศษ และ Event Bonus

2

Class Evolution

ทุก 3 EXP = เลเวลอัป → เลือก Evolution Class จาก 18 สายอาชีพ

3

Stat Allocation

แจก Point ให้ STR / DEX / VIT / INT ตามสไตล์การเล่นของตัวเอง

4

Secret Mission

ทุกฝ่ายมีภารกิจลับที่ต้องสำเร็จก่อน Turn 10 เพื่อชนะ

Chapter Four

Card System

ระบบไพ่ทั้งหมด
⚔️
อาวุธ / เกราะ
ไพ่อุปกรณ์สำหรับเพิ่มพลัง ป้องกัน และ Effect พิเศษ — มี 4 ระดับ: พื้นบ้าน / หวงห้าม / ลี้ลับ / สมบัติสวรรค์
🌪️
เหตุการณ์
ไพ่ที่ส่งผลต่อผู้เล่นทุกคนหรือบางคน — แบ่งเป็น 3 ประเภทตามช่วงเวลาที่ใช้งาน
🕳️
กับดัก
ไพ่วางซ่อนบนแผนที่ ศัตรูที่เดินผ่านโดนผลทันที — มีกว่า 50 รูปแบบ
🔮
เวทย์มนตร์
ไพ่พลังเวทสำหรับโจมตี ป้องกัน ฮีล และควบคุมสนามรบ

🃏 Card Rarity System

ระดับ (Rarity) สัญลักษณ์ Drop Rate ตัวอย่าง ข้อจำกัดพิเศษ
พื้นบ้าน (Common)⚪50%มีดลอบสังหาร, ขวานสองคมไม่มี
หวงห้าม (Rare)🔵30%ธนูคริสตัลน้ำแข็ง, เกราะพรางเงาเงื่อนไขสถานที่/เวลา
ลี้ลับ (Epic)🟣15%โล่ปราการดวงดาว, ดาบสิ้นแสงศักดิ์ใช้ได้ครั้งจำกัด
สมบัติสวรรค์ (Legendary)🟡5%ค้อนราชันย์สังหาร, โล่แห่งปฐมกาลครั้งเดียวต่อเกม

Chapter Five

World Design

การออกแบบโลกและแผนที่
🏰
พระราชวัง (Palace Zone)
ศูนย์กลางอำนาจ — พระราชาได้ +HP ที่นี่ กบฏถ้าบุกถึงจะทริก Event ใหญ่ มี "ศาลบัลลังก์" เป็น Special Zone
🏘️
หมู่บ้าน (Village Zone)
ราษฎรเริ่มต้นที่นี่ มีตลาดกลางเมือง ซื้อไอเท็มราคาครึ่งหนึ่ง เกิด Event "ไฟไหม้" หรือ "โรคระบาด" ได้บ่อย
🌲
ป่าลึก (Forest Zone)
ซ่อนตัวได้ด้วยการ์ด Stealth ป่าเวทมนตร์มี Bonus EXP จากสัตว์ร้าย มีโอกาส Spawn มอนสเตอร์
⛺
ค่ายกบฏ (Rebel Camp)
ฐานทัพกบฏ — อัพเกรด Class ได้เร็วขึ้น มีตลาดมืดซื้อของหายาก แต่ถ้าพระราชาบุกถึงจะเกิด Event "ไฟไหม้ค่าย"
🏔️
ถ้ำมังกร (Dragon Cave)
พื้นที่ High Risk High Reward — มี Boss มังกร ทอยเต๋าผ่าน เพื่อรับสมบัติ Legendary รับ +3 Gold แต่เสี่ยง HP
🌫️
Zone พิเศษ (Secret Areas)
ค่ายลับคนทรยศ / หอคอยเวทโบราณ / สุสานนักรบ / กระโจมหมอดู — Unlock เมื่อ Turn 4 ขึ้นไป

🐉 Boss Encounters

BossLocationHPAttackReward
🐍 งูเขียวยักษ์ (Jade Serpent)ป่าลึก15โจมตี 3 ติดพิษCard Legendary + 5 Gold
🐺 หมาป่าดำ (Shadow Wolf)ชายป่า12โจมตีซ้ำ 2 ครั้งWeapon Epic + 3 EXP
🐲 มังกรเพลิง (Fire Dragon)ถ้ำมังกร25AOE ไฟ 5 ทุกคนLegendary Weapon + Trophy
💀 โครงกระดูกราชา (Skeleton King)สุสาน18สาป -2 STR ถาวรCard Rare x3 + EXP 5

Chapter Six — Technical Design

Unity Development Pipeline

แนวทางการพัฒนาด้วย Unity อย่างละเอียด
🎯 Engine Choice: Unity 6 LTS (Universal Render Pipeline) — รองรับ PC / Mobile / Console จาก Codebase เดียว, ระบบ Networking ผ่าน Unity Netcode for GameObjects, รองรับ 2D Spine Animation สำหรับ Stickman Art Style

🏗️ Project Structure ใน Unity

Assets/
├── 📁 _Art/
│   ├── Stickman/    ← Spine2D .skel + Atlas ของตัวละคร
│   ├── Cards/      ← Card artwork PNG (transparent)
│   ├── UI/         ← Parchment UI theme assets
│   └── Map/       ← Tilemap sprites, Zone art
├── 📁 _Scripts/
│   ├── Core/      ← GameManager, TurnManager, EventBus
│   ├── Cards/     ← CardData (ScriptableObject), CardController
│   ├── Characters/ ← CharacterData, StatSystem, ClassEvolution
│   ├── Board/     ← GridManager, TileData, ZoneEffect
│   ├── Combat/    ← CombatResolver, DiceRoller, DamageCalc
│   ├── Network/   ← LobbyManager, SyncManager (Netcode)
│   └── UI/       ← CardHandUI, PlayerHUDUI, MapUI
├── 📁 _Data/       ← ScriptableObject Database ทั้งหมด
├── 📁 _Prefabs/    ← Player, Card, Tile, Effect prefabs
└── 📁 _Scenes/
    ├── MainMenu.unity
    ├── Lobby.unity
    └── GameBoard.unity

📦 ขั้นตอนที่ 1 — Setup โปรเจค

1

ติดตั้ง Unity 6 LTS

ดาวน์โหลด Unity Hub → เลือก Unity 6000.x.x LTS → เลือก Module: WebGL / Android / iOS / PC ตามแพลตฟอร์มเป้าหมาย

2

สร้าง Project ใหม่

Template: 2D (URP) → ตั้งชื่อ ShadowOfThrone → ตั้งค่า Color Space เป็น Linear, Target Frame Rate 60fps

3

Import Packages จาก Package Manager

Netcode for GameObjects Spine-Unity TextMeshPro Unity Gaming Services Addressables

4

ตั้งค่า Assembly Definitions

สร้าง .asmdef แยกแต่ละ Module (Core / Cards / Network) เพื่อลด Compile Time และป้องกัน Circular Dependency

🎴 ขั้นตอนที่ 2 — Card System

1

สร้าง CardData ScriptableObject

สร้าง Class CardData : ScriptableObject มี Field: cardName, cardType, rarity, effect, condition, artwork, cost

2

สร้าง CardDatabase

ScriptableObject CardDatabase เก็บ List ของ CardData ทุกใบ (อาวุธ 50+ / เหตุการณ์ 50+ / กับดัก 50+ / เวทย์ 50+) รวม 200+ ใบ

3

CardController ใน Scene

Component ที่ render Card UI, จัดการ Drag-and-Drop, ส่ง Event ไปยัง GameManager เมื่อผู้เล่นใช้ไพ่

4

Effect System

ใช้ Pattern Command + Strategy — แต่ละ Effect เป็น Class แยก implement Interface ICardEffect.Execute() เพื่อ Extensibility

🗺️ ขั้นตอนที่ 3 — Board System

1

Tilemap Setup

ใช้ Unity Tilemap (Isometric) → สร้าง Tile Palette สำหรับ Palace / Village / Forest / Camp Zone แต่ละประเภทมี TileData Script แนบ

2

GridManager

จัดการ Grid Logic, Movement Range, Path Highlighting, Trap Placement โดยใช้ A* Pathfinding (AStarPathfindingProject package)

3

Zone Effects

แต่ละ Zone มี ZoneEffect Component ที่ trigger เมื่อ Player เข้า/ออก — ใช้ Physics2D Trigger Collider

🌐 ขั้นตอนที่ 4 — Multiplayer (6 Players)

1

Unity Relay + Lobby Service

ใช้ Unity Gaming Services → Lobby สร้าง/เข้าห้อง → Relay เชื่อมต่อ P2P ผ่าน Server โดยไม่ต้องมี Dedicated Server

2

NetworkVariable Sync

Sync ข้อมูล: Player Position, HP, Card Hand (Hidden/Visible), Turn Order, Board State ผ่าน NetworkVariable<T>

3

ServerRPC / ClientRPC

การกระทำทุกอย่าง (ใช้ไพ่, โจมตี, เดิน) ส่งผ่าน ServerRPC → Host ตรวจสอบ → ส่งผล ClientRPC ไปทุกคน

🎨 ขั้นตอนที่ 5 — Stickman Animation (Spine2D)

✏️
วาด Concept Art
วาด Stickman แต่ละ Class ใน Procreate หรือ Clip Studio — Ink Style, เส้นหยาบ, เน้น Silhouette ชัด ดังเห็นในภาพ Concept ที่ส่งมา
💀
ริก Spine2D
แยกชิ้นส่วน (Head / Body / Arms / Legs) ใน Photoshop → Import เข้า Spine2D → ทำ Skeleton → Animate (Idle / Attack / Die / Evolve)
🎮
Export เข้า Unity
Export Spine2D เป็น .skel + .atlas → Import ผ่าน Spine-Unity Runtime → ใช้ SkeletonAnimation Component ควบคุม Animation State

Chapter Eight

Development Roadmap

แผนการพัฒนา
Phase 0 — Pre-Production (เดือน 1–2)
Prototype & Validation
ทดสอบ Core Loop ด้วย Physical Board Game จริง → ปรับสมดุล Card / Combat / Turn → ทำ Digital Prototype ใน Unity (2D กระดาน + ไพ่ Local Play)
Phase 1 — MVP (เดือน 3–5)
Minimum Viable Product
Local Multiplayer 2–4 คน บน PC → 4 Role + 6 Class + 80 Cards → 1 แผนที่ → Combat / Dice / Turn ทำงานได้ → Stickman Animation ขั้นพื้นฐาน (Idle / Attack)
Phase 2 — Alpha (เดือน 6–9)
Core Systems Complete
Online Multiplayer 6 คน (Unity Relay) → Class Evolution 18 คลาส → Cards 150+ ใบ → 3 Zone แผนที่ → Boss Encounters → Quest System → Internal Testing ทีม + นักเล่นทดสอบ 20 คน
Phase 3 — Beta (เดือน 10–13)
Closed / Open Beta
Cards 200+ ใบ → Endgame Content → Battle Pass Season 1 → Mobile Port (Android / iOS) → Performance Optimization → Closed Beta 500 คน → Open Beta 5,000 คน → Bug Fix รอบใหญ่
Phase 4 — Full Release (เดือน 14+)
Launch & Live Service
Launch บน PC (Steam / Epic) + Mobile → Seasonal Content อัปเดตทุก 8 สัปดาห์ → Console Port (Nintendo Switch / PS5) เดือน 18 → Tournament Mode → Story Campaign DLC เดือน 24

Riskระดับแนวทางแก้ไข

Multiplayer Sync Bugสูงใช้ Authoritative Server, Unit Test Network Layer อย่างละเอียด
Art Style ใช้เวลานานกลางเริ่มจาก Placeholder → Hire Spine Artist พาร์ทไทม์ใน Phase 2
ผู้เล่นไม่เข้าใจ Hidden RoleกลางInteractive Tutorial ใน Scene แยก, Hint System
งบประมาณ Developmentต่ำUnity Personal License (ฟรี), Indie Team 3–5 คน, เน้น Asset ทำเอง
